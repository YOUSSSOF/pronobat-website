# نقش‌های سفارشی و دسترسی‌ها

در این راهنما یک نقش سفارشی وردپرس می‌سازیم که به کارمند اجرایی دسترسی کافی برای مدیریت نوبت‌ها بدهد، اما نه به تنظیمات حساس افزونه.

## سناریو

یک منشی دارید که باید بتواند:
- ✅ لیست نوبت‌ها را ببیند
- ✅ وضعیت نوبت را تغییر دهد
- ✅ گزارش‌های اولیه را ببیند
- ❌ تنظیمات افزونه را ویرایش نکند
- ❌ درگاه‌های پرداخت را تغییر ندهد
- ❌ کارمندان و سرویس‌ها را حذف نکند

---

## مرحله ۱: ساخت نقش سفارشی

کد زیر را در فایل `functions.php` تم خود (یا یک افزونه سفارشی) اضافه کنید:

```php
/**
 * ساخت نقش «منشی نوبت» با دسترسی‌های محدود
 * این تابع یک بار اجرا می‌شود (هنگام فعال‌سازی تم یا افزونه)
 */
function my_create_pronobat_receptionist_role(): void {
    // اگر نقش از قبل وجود داشت، چیزی نساز
    if ( get_role( 'pronobat_receptionist' ) ) {
        return;
    }

    add_role(
        'pronobat_receptionist',
        'منشی نوبت‌دهی',
        [
            // دسترسی‌های پایه وردپرس
            'read'                         => true,

            // دسترسی‌های نوبت
            'pronobat_manage_appointments'   => true,
            'pronobat_view_reports'          => true,

            // دسترسی به مدیریت سرویس‌ها (فقط مشاهده و ویرایش، نه حذف)
            'pronobat_manage_services'       => true,
        ]
    );
}
add_action( 'init', 'my_create_pronobat_receptionist_role' );
```

<Callout type="info">
تابع `add_role` فقط یک بار نیاز به اجرا دارد. وردپرس نقش‌ها را در پایگاه داده ذخیره می‌کند، پس نیازی به تکرار نیست.
</Callout>

## مرحله ۲: اضافه کردن دسترسی به نقش موجود

اگر می‌خواهید دسترسی پرونوبت را به یک نقش موجود (مثلاً Editor) بدهید:

```php
function my_grant_pronobat_caps_to_editor(): void {
    $role = get_role( 'editor' );
    if ( ! $role ) {
        return;
    }

    $role->add_cap( 'pronobat_manage_appointments' );
    $role->add_cap( 'pronobat_view_reports' );
}
add_action( 'init', 'my_grant_pronobat_caps_to_editor' );
```

## مرحله ۳: اختصاص نقش به کاربر

<Steps>

### رفتن به مدیریت کاربران

در وردپرس، به **کاربران ← همه کاربران** بروید.

### ویرایش کاربر

روی نام کاربر منشی کلیک کنید.

### تغییر نقش

فیلد **نقش** را به **منشی نوبت‌دهی** تغییر دهید.

### ذخیره

**به‌روزرسانی کاربر** را بزنید.

</Steps>

## جدول کامل دسترسی‌ها

| Capability | توضیح | پیشنهاد برای منشی |
|---|---|---|
| `pronobat_manage_appointments` | مشاهده و مدیریت نوبت‌ها | ✅ بده |
| `pronobat_manage_settings` | تنظیمات افزونه | ❌ نده |
| `pronobat_manage_staff` | افزودن/ویرایش کارمندان | بستگی دارد |
| `pronobat_manage_services` | افزودن/ویرایش سرویس‌ها | بستگی دارد |
| `pronobat_view_reports` | مشاهده گزارش‌ها | ✅ بده |
| `pronobat_process_refunds` | درخواست استرداد وجه | ❌ نده |

## پاک کردن نقش (در صورت نیاز)

```php
remove_role( 'pronobat_receptionist' );
```
