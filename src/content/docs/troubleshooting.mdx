# عیب‌یابی

## فعال کردن حالت دیباگ

قبل از هر چیز، حالت دیباگ را فعال کنید تا لاگ‌های دقیق‌تری داشته باشید:

```php
// wp-config.php
define( 'WP_DEBUG', true );
define( 'WP_DEBUG_LOG', true );
define( 'WP_DEBUG_DISPLAY', false );
define( 'PRONOBAT_DEBUG', true );
```

سپس لاگ را دنبال کنید:

```bash
tail -f wp-content/debug.log
```

## بازه‌های زمانی نمایش داده نمی‌شوند

**۱. ساعت کاری تنظیم نشده:**
پرونوبت ← کارمندان → کارمند موردنظر → ساعت کاری را تنظیم کنید.

**۲. کش منقضی نشده:**
Object Cache را پاک کنید یا `pronobat_cache_ttl` را به `0` تنظیم کنید.

**۳. تعارض با افزونه کش:**
Page Caching برای endpoint های REST پرونوبت را غیرفعال کنید. در W3TC/WP Super Cache مسیر `/wp-json/pronobat/` را exclude کنید.

**۴. تاریخ گذشته است:**
`min_booking_lead_time` در تنظیمات عمومی را بررسی کنید.

---

## پیامک ارسال نمی‌شود

**۱. API Key اشتباه:**
دکمه «ارسال پیامک آزمایشی» را بزنید و خطا را بررسی کنید.

**۲. اعتبار ناکافی:**
موجودی حساب سرویس پیامک را بررسی کنید.

**۳. opt-out فعال است:**
در **پرونوبت ← مشتریان** بررسی کنید که مشتری گزینه opt-out را فعال نکرده باشد.

**۴. سقف روزانه رسیده:**
مقدار `sms_daily_cap` را در تنظیمات پیامک افزایش دهید.

**۵. فرمت شماره اشتباه:**
شماره باید دقیقاً `09XXXXXXXXX` (۱۱ رقم) باشد.

---

## پرداخت ناموفق یا Redirect اشتباه

**۱. URL بازگشت ثبت نشده:**
آدرس `/wp-json/pronobat/v1/payments/verify/{gateway}` را در پنل درگاه ثبت کنید.

**۲. IP سرور مسدود است:**
برخی درگاه‌ها (مثل زرین‌پال) نیاز به whitelist کردن IP سرور دارند.

**۳. Merchant ID اشتباه:**
در محیط sandbox از credential های sandbox استفاده کنید.

---

## REST API پاسخ نمی‌دهد

**۱. Permalink ها ریست نشده:**
**تنظیمات ← پیوندهای یکتا** را باز و ذخیره کنید.

**۲. mod_rewrite غیرفعال:**
مطمئن شوید `AllowOverride All` در تنظیمات Apache فعال است.

---

## خطای ۴۰۳ در مسیرهای مدیریتی

Nonce منقضی شده. صفحه را رفرش کنید تا Nonce تازه دریافت شود. عمر پیش‌فرض Nonce در وردپرس ۱۲ ساعت است.

---

## یادآورها ارسال نمی‌شوند

**۱. WP-Cron غیرفعال:**
بررسی کنید `DISABLE_WP_CRON` در `wp-config.php` مقدار `true` ندارد.

**۲. Cron بسیار کم اجرا می‌شود:**
برای سایت‌های کم‌ترافیک، یک Real Cron از طریق cPanel یا سرور تنظیم کنید:
```bash
*/5 * * * * wget -q -O - https://example.com/wp-cron.php?doing_wp_cron
```
