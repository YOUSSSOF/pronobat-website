# GDPR و حریم خصوصی

## داده‌های ذخیره‌شده

| جدول | داده‌های شخصی | مدت نگهداری پیش‌فرض |
|---|---|---|
| `wp_pronobat_appointments` | نام، تلفن، ایمیل مشتری | ۳ سال |
| `wp_pronobat_payments` | مبلغ، شماره مرجع | ۷ سال (الزام مالیاتی) |
| `wp_pronobat_customers` | نام، تلفن، ایمیل، opt-out | تا درخواست حذف |
| `wp_pronobat_logs` | IP، User-Agent (فقط خطا) | ۹۰ روز |

## درخواست صادر کردن داده

کاربران می‌توانند از طریق ابزار بومی وردپرس «درخواست صادر کردن داده شخصی» درخواست دهند. پرونوبت به‌طور خودکار داده‌های خود را در پاسخ به این درخواست‌ها اضافه می‌کند.

همچنین از طریق API:

```
GET /wp-json/pronobat/v1/admin/customers/{id}/export
```

**دسترسی:** `pronobat_manage_settings` + `X-WP-Nonce`

پاسخ: JSON با تمام نوبت‌ها و اطلاعات مشتری در یک فایل قابل دانلود.

## ناشناس‌سازی (Anonymize)

```
POST /wp-json/pronobat/v1/admin/customers/{id}/anonymize
```

**دسترسی:** `pronobat_manage_settings`

این endpoint:

1. نام را با `Customer #ID` جایگزین می‌کند
2. تلفن را با `XXXXXXXXXXX` جایگزین می‌کند
3. ایمیل را با `anonymized@pronobat.invalid` جایگزین می‌کند
4. داده‌های گوگل تقویم را قطع می‌کند
5. لاگ‌های مرتبط را حذف می‌کند

نوبت‌های گذشته (**تاریخچه مالی**) برای اهداف حسابداری بدون اطلاعات شخصی حفظ می‌شوند.

## تنظیمات نگهداری

مسیر: **پرونوبت ← تنظیمات ← پیشرفته ← نگهداری داده**

| تنظیم | پیش‌فرض |
|---|---|
| نگهداری اطلاعات نوبت | ۳ سال |
| نگهداری لاگ خطا | ۹۰ روز |
| حذف داده‌ها هنگام حذف افزونه | غیرفعال |

<Callout type="warning">
اگر «حذف داده‌ها هنگام حذف افزونه» فعال باشد، با حذف افزونه، **تمام جداول و داده‌ها** به‌طور غیرقابل‌بازگشت حذف می‌شوند.
</Callout>

## Privacy Policy

متن پیشنهادی برای صفحه سیاست حریم خصوصی سایت شما در **پرونوبت ← ابزارها ← حریم خصوصی** قابل مشاهده است.
