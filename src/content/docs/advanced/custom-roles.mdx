# نقش‌ها و دسترسی‌ها

## ثابت‌های Capability

| Capability | توضیح |
|---|---|
| `pronobat_manage_appointments` | مشاهده، ویرایش، و تغییر وضعیت نوبت‌ها |
| `pronobat_manage_settings` | ویرایش تنظیمات افزونه |
| `pronobat_manage_staff` | افزودن و ویرایش کارمندان |
| `pronobat_manage_services` | افزودن و ویرایش خدمات |
| `pronobat_view_reports` | مشاهده گزارش‌ها و داشبورد |
| `pronobat_process_refunds` | درخواست استرداد وجه |

## نقش‌های پیش‌فرض

| نقش وردپرس | Capability های پرونوبت |
|---|---|
| `administrator` | همه |
| `pronobat_manager` | همه به جز تنظیمات سیستمی |
| `pronobat_staff` | فقط نوبت‌های خود + `pronobat_manage_appointments` |

## اعطای دسترسی به نقش سفارشی

برای دادن دسترسی پرونوبت به نقش دلخواه خود:

```php
add_action( 'after_setup_theme', function() {
    $role = get_role( 'editor' ); // نام نقش خود را اینجا وارد کنید
    if ( ! $role ) return;

    $role->add_cap( 'pronobat_manage_appointments' );
    $role->add_cap( 'pronobat_view_reports' );
} );
```

<Callout type="warning">
این کد را فقط یک بار اجرا کنید (مثلاً در `functions.php` تم یا یک افزونه). به دلیل ذخیره‌شدن capability در پایگاه داده، هر بار اجرا بی‌اثر است اما نیاز به اجرای مجدد ندارد.
</Callout>

## سلب دسترسی

```php
$role = get_role( 'author' );
if ( $role ) {
    $role->remove_cap( 'pronobat_manage_appointments' );
}
```

## بررسی دسترسی در کد سفارشی

```php
if ( current_user_can( 'pronobat_manage_appointments' ) ) {
    // نمایش دکمه مدیریت
}
```

## امنیت REST API

هر route مدیریتی پرونوبت دارای `permission_callback` است:

```php
'permission_callback' => function() {
    return current_user_can( 'pronobat_manage_appointments' );
},
```

پنهان کردن دکمه‌ها در رابط کاربری **جایگزین** بررسی permission سرور‌سایدی نیست.
