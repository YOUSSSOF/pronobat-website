# تنظیمات پیشرفته

مسیر: **پرونوبت ← تنظیمات ← پیشرفته**

## حالت دیباگ

وقتی **Debug Mode** فعال باشد، پرونوبت لاگ‌های دقیق‌تری به `wp-content/debug.log` می‌نویسد:

- درخواست‌های REST API ورودی
- فراخوانی‌های API پرداخت و پاسخ‌ها
- ارسال پیامک (بدون نمایش محتوا)
- خطاهای cron

<Callout type="warning">
حالت دیباگ را در محیط تولید فعال نگذارید. لاگ‌ها ممکن است اطلاعات حساس داشته باشند.
</Callout>

برای فعال کردن از طریق `wp-config.php`:

```php
define( 'PRONOBAT_DEBUG', true );
define( 'WP_DEBUG', true );
define( 'WP_DEBUG_LOG', true );
define( 'WP_DEBUG_DISPLAY', false );
```

## کش (Slot Cache)

زمان‌بندی موجود در Object Cache وردپرس ذخیره می‌شود.

| تنظیم | پیش‌فرض | توضیح |
|---|---|---|
| Cache TTL | ۳۰۰ ثانیه | مدت ماندگاری کش slot |
| پاک کردن کش | خودکار | هنگام ذخیره ساعت کاری یا نوبت جدید |

برای پاک کردن دستی کش:

```php
do_action( 'pronobat_flush_slot_cache' );
```

## محدودیت نرخ (Rate Limiter)

مسیرهای عمومی REST به‌صورت پیش‌فرض محدود می‌شوند:

| مسیر | حد |
|---|---|
| `GET /slots` | ۱۰ درخواست در دقیقه به ازای IP |
| `POST /appointments` | ۳ درخواست در دقیقه به ازای IP |

برای تغییر حد:

```php
add_filter( 'pronobat_rate_limit', function( int $limit, string $route ) {
    if ( $route === 'slots' ) return 20;
    return $limit;
}, 10, 2 );
```

## CSS سفارشی

کدی که در این فیلد وارد می‌کنید مستقیماً به `<head>` صفحاتی که شورت‌کد نوبت دارند اضافه می‌شود. برای سفارشی‌سازی ظاهر فرم رزرو بدون ویرایش فایل‌های پوشه استفاده کنید.

| تنظیم | پیش‌فرض |
|---|---|
