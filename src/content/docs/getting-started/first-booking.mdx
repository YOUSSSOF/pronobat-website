# اولین نوبت

این راهنما چرخه کامل یک نوبت را از دید مشتری و از دید مدیر توضیح می‌دهد.

## از دید مشتری

### مرحله ۱ — انتخاب خدمت

مشتری وارد صفحه رزرو می‌شود و خدمت مورد نظر را انتخاب می‌کند. اگر چند کارمند وجود داشته باشد، در مرحله بعد می‌تواند کارمند خاصی انتخاب کند یا «هر کارمند» را بزند.

### مرحله ۲ — انتخاب تاریخ و ساعت

تقویم جلالی ساعت‌های خالی را به‌صورت بلادرنگ نمایش می‌دهد. ساعت‌هایی که قبلاً رزرو شده‌اند یا خارج از ساعت کاری هستند، خاکستری و غیرقابل انتخاب‌اند.

### مرحله ۳ — اطلاعات تماس

مشتری نام، شماره موبایل (برای پیامک)، و ایمیل (اختیاری) را وارد می‌کند. نیاز به ثبت‌نام نیست.

### مرحله ۴ — پرداخت

اگر خدمت نیاز به پرداخت داشته باشد، مشتری به درگاه پرداخت هدایت می‌شود. پس از تأیید پرداخت، سرور وضعیت را بررسی می‌کند.

<Callout type="warning">
هیچ‌گاه وضعیت پرداخت از سمت مرورگر پذیرفته نمی‌شود. تأیید همیشه از طریق API درگاه به سرور انجام می‌شود.
</Callout>

### مرحله ۵ — تأیید

مشتری صفحه تأیید را می‌بیند و پیامک تأییدیه دریافت می‌کند:

> نوبت شما با موفقیت ثبت شد.
> خدمت: کوتاهی مو — کارمند: علی — ۱۴۰۵/۰۳/۰۱ ساعت ۱۱:۰۰

## از دید مدیر

### دریافت اعلان

بلافاصله پس از ثبت نوبت، مدیر اعلان ایمیل یا پیامک دریافت می‌کند (بسته به تنظیمات).

### مشاهده نوبت

در **پرونوبت ← نوبت‌ها** نوبت جدید با وضعیت مناسب ظاهر می‌شود:

| وضعیت | معنا |
|---|---|
| `تأییدشده` | پرداخت انجام شده یا خدمت رایگان بود |
| `در انتظار پرداخت` | لینک پرداخت ارسال شده، هنوز پرداخت نشده |
| `در انتظار تأیید` | نیاز به تأیید دستی مدیر دارد |

### تغییر وضعیت دستی

روی نوبت کلیک کنید تا جزئیات باز شود. می‌توانید:
- وضعیت را تغییر دهید (تأیید / لغو / انجام‌شده)
- یادداشت اضافه کنید
- پیامک یادآور دستی بفرستید
- استرداد وجه درخواست کنید

## رویدادهای قابل hook

```php
// بعد از ثبت نوبت
add_action( 'pronobat_appointment_created', function( int $id, array $appt ) {
    // ارسال به CRM سفارشی
    my_crm_sync( $appt );
}, 10, 2 );

// بعد از تأیید پرداخت
add_action( 'pronobat_payment_received', function( int $id, array $payment ) {
    // ثبت در سیستم حسابداری
    my_accounting_record( $payment );
}, 10, 2 );
```

برای فهرست کامل hook‌ها به [مستندات هوک‌ها](/docs/hooks) مراجعه کنید.
