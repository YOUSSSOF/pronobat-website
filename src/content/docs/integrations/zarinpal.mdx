# یکپارچه‌سازی زرین‌پال

## پیش‌نیازها

- حساب کاربری در [zarinpal.com](https://zarinpal.com)
- کد Merchant از بخش «فروشگاه‌ها»

## پیکربندی در نوبت

مسیر: **پرونوبت ← تنظیمات ← پرداخت ← زرین‌پال**

| فیلد | توضیح |
|---|---|
| Merchant ID | کد ۳۶ کاراکتری |
| حالت | `sandbox` برای آزمایش، `production` برای واقعی |
| آدرس بازگشت | به‌طور خودکار پر می‌شود |

## حالت Sandbox

در حالت Sandbox:
- Merchant ID می‌تواند هر مقداری باشد
- پرداخت واقعی انجام نمی‌شود
- برای آزمایش از کارت شماره `6219861034529007` استفاده کنید

## URL بازگشت

پرونوبت به‌طور خودکار آدرس بازگشت را تنظیم می‌کند:

```
https://example.com/wp-json/pronobat/v1/payments/verify/zarinpal
```

این آدرس را در پنل زرین‌پال ثبت کنید.

## خطاهای رایج

| کد خطا | علت | راه‌حل |
|---|---|---|
| `-9` | اعتبارسنجی ناقص | Merchant ID را بررسی کنید |
| `-10` | IP غیرمجاز | IP سرور را در پنل زرین‌پال whitelist کنید |
| `-21` | هیچ تراکنش مالی پیدا نشد | کاربر از درگاه برنگشته یا Authority منقضی شده |
| `-22` | تراکنش ناموفق | کاربر پرداخت را لغو کرده |
| `100` | موفق | — |
| `101` | قبلاً تأیید شده | نوبت از قبل پرداخت شده (نادیده بگیرید) |

<Callout type="info">
در صورت دریافت کد `101`، نوبت را تأیید‌شده فرض کنید. این حالت زمانی اتفاق می‌افتد که کاربر صفحه موفقیت را رفرش کرده باشد.
</Callout>
